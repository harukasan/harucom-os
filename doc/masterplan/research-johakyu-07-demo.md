# research 07: 統合・演出・本番ランブック (M10)

## 目的

全要素 (DMX エンジン / オーディオ / パターンコア / DSL / UI / 2 台のムービングライト) を
統合し、関西Ruby会議の能舞台で「音と光が同期した演出をライブコーディングで見せる」デモを
仕上げる。演出プリセットを用意し、本番で破綻しない運用手順を固める。

## 前提

- 先に読む: [masterplan-johakyu.md](masterplan-johakyu.md) と research 00-06 全て。
- M9 まで完了 (UI でライブコーディングでき、音光が同期駆動)。
- 実機一式 (Harucom + M5 DMX Unit + SHEHDS Spot 80W 2 台 + 音声出力) が揃っていること。

## 対象マイルストーン

- M10: デモ仕上げ。完了条件 = 全 2 台が音と同期して演出し、ライブ編集が破綻しない。

## 設計詳細

### 統合構成

- Harucom 1 台 → Grove(J5) → M5 DMX Unit → XLR → SHEHDS Spot 80W 2 台のデイジーチェーン。
  最終灯体で終端 120Ω 有効。
- 音声は 3.5mm ジャックから会場 PA へ。
- `rootfs/app/johakyu.rb` を起動し、USB キーボードでライブコーディング。DVI を会場
  スクリーンに出してユニバース可視化を観客に見せる。

### 演出プリセット

`rootfs/data/johakyu/` 等に複数のデモパターン (.rb) を用意し、本番で読み込む。例:

- 序 (jo): ゆっくりした `sine.slow` のパン振り + 低密度のキック。静かな立ち上がり。
- 破 (ha): ミニ記法のドラム + `dmx(:smalls).color` の色変化 + dimmer の `saw.segment`。
- 急 (kyu): `euclid` ストロボ + `fast` でテンポ感増し + 全灯 (`dmx(:all)`) のクライマックス。

任意で jo/ha/kyu アレンジ区分機能 (research 05) を使い、1 つのスクリプトで山場を構築。

### 演出設計上の注意

- dimmer/strobe (電気応答=速い) を音同期の主役にし、pan/tilt (機械=遅い) はゆっくりした
  signal に割り当てる (R14)。
- 観客に「DMX = ch 番号 × 値」が伝わるよう、UI のユニバース可視化と実際の灯体の動きを対応
  させて見せる (ch を 1 つ動かす → 画面のセルが光る → 灯体が動く、の対応を実演)。

### アドレス整合 (本番前必須)

各灯体の物理 DMX アドレスを `fixture.rb` の patch (research 03) と一台ずつ一致させ、一致表
(灯体ラベル ↔ base ch ↔ 物理位置) を本番セットでも同じ並びにする。最も間違えやすい工程。

### 本番ランブック

**前提: 灯体は DMX 信号断で自動消灯しない (最後の値で固まる)**。消灯は常にこちらから明示する。

- 起動順: 灯体電源 → M5 ユニット → Harucom (DVI/音声/キーボード接続確認) → `johakyu.rb`
  (起動直後にゼロフレームが出て前回の固まりが消える)。
- 安全: アプリは `DMX.keepalive` を毎ループ送り、ハング時は C 側 dead-man が自動消灯
  (research 01)。例外時は `rescue` で `DMX.blackout`。
- フォールバック: コンパイルエラーでも走行継続 (last-good)。最悪時は `DMX.blackout` →
  プリセット再読込。Ctrl-Alt-Del での再起動手順も確認 (再起動でゼロフレーム → 消灯)。
- 終了: `DMX.blackout` → アプリ終了 → 灯体電源 OFF。

## 調査項目

### R13: デイジーチェーン終端 / 反射 (全系) (影響: 中)

research 00/03 では少数台で確認済み。本番の全 2 台直列で最終確認する。

- 検証: 全 2 台を本番ケーブル長で接続し、最遠灯体まで値が化けないこと、終端の有無で挙動が
  変わらないことを確認。本番と同じ active_slots / リフレッシュ周波数で長時間 (数十分) 流し、
  ちらつき・コマ落ち・アンダーランが無いか確認。

### R14: 音と光の知覚同期ズレ (影響: 低)

- 検証: 本番演出で dimmer/strobe 主体の同期が観客視点で「揃って見える」か確認。pan/tilt の
  遅延が気になる箇所は signal を緩める。

### R19: 全系での消灯保証 (影響: 高)

- 検証: 本番構成 (2 台) で、アプリ例外/ハング/正常終了/再起動のいずれでも全灯が消灯すること。
  起動直後のゼロフレームで前回の固まりが消えること。keepalive 停止 → dead-man 消灯を実演で確認。

## 受け入れ条件 (DoD)

- 2 台全てが意図どおりに音と同期して動く。
- 序破急のプリセットで「静→展開→急」の山場が成立する。
- 長時間運用で DMX/音声のアンダーランや UI のコマ落ちが無い。
- ライブ編集中にエラーを出しても演出が止まらない (last-good フォールバック)。
- 例外/ハング/正常終了/再起動のいずれでも全灯が消灯する (R19)。
- 物理アドレスと patch が一致表どおりに整合している。
- 起動/フォールバック/終了のランブックが文書化され、本番前にリハーサル済み。
